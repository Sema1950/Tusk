# Tusk Agent Instructions

This repository uses `instructions.md` as the primary working instruction file.

Before making project changes, read:

* `instructions.md`
* `docs/current_state.md`
* `docs/backlog.md`
* `docs/project_overview.md` when the task depends on purpose, scope, or boundaries

Follow the file-responsibility rules in `instructions.md`.

Do not invent missing requirements, technologies, business rules, workflows, schemas, roles, validations, integrations, or implementation details.

Keep deterministic processing, removable AI assistance, and human judgment clearly separated.

Use the approved project files as the source of truth. If project files conflict, identify the conflict instead of silently choosing one.

Make only the changes requested by the user. Do not expand scope or continue into the next step without approval when the user asks for step-by-step work.

Do not commit confidential customer information, production insurance data, secrets, passwords, API keys, tokens, or private certificates.
